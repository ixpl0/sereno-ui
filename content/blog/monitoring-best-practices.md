# 5 лучших практик мониторинга инфраструктуры

Мониторинг — это не просто сбор метрик. Это искусство получать нужную информацию в нужное время.

## 1. Не мониторьте всё подряд

Больше метрик ≠ лучше. Сфокусируйтесь на том, что действительно влияет на пользователей:

- Время ответа API
- Доступность сервисов
- Ошибки в логах

## 2. Настройте умные пороги

Статические пороги — это прошлый век. Используйте динамические алерты, которые учитывают:

- Время суток
- День недели
- Исторические паттерны

## 3. Группируйте связанные алерты

Один сбой может вызвать каскад из десятков алертов. Группировка поможет увидеть корень проблемы.

## 4. Документируйте runbooks

Каждый алерт должен содержать ссылку на инструкцию: что проверить, как починить, к кому обратиться.

## 5. Регулярно пересматривайте алерты

Алерт, который игнорируют — бесполезен. Раз в месяц удаляйте неактуальные и настраивайте проблемные.
